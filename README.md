# SSHX 远程终端 · GitHub Actions 一键部署

这是一个基于 **GitHub Actions + SSHX** 的临时远程终端方案，无需自己搭建服务器，只需手动触发一次工作流，即可在几分钟内获得一个带完整编译环境的 Ubuntu 终端。

---

## 🚀 使用方法

1.  进入本仓库的 **Actions** 页面
2.  找到对应工作流，点击 `Run workflow` 按钮，手动启动一次任务
3.  等待工作流自动初始化并配置环境（约 5-10 分钟）
4.  当终端日志中出现 `sshx.io/` 格式的链接时，复制该链接到浏览器打开
5.  第一次进入时可能不显示终端：
    - 按提示设置用户名
    - 点击页面上的 `+` 号，即可创建并打开终端窗口

---

## ⚠️ 重要声明与注意事项

- 本工具仅可用于**合法合规用途**，**禁止用于任何违规、侵权或非法活动**。
- 工作流超时时间设置为 360 分钟，任务结束后终端将自动销毁，不会留存任何数据。
- 请勿公开分享你的 SSHX 链接，避免被他人滥用。
- 本方案仅为学习交流用途，因违规使用产生的一切后果由使用者自行承担。
- 建议在网络环境良好（如有代理）的情况下使用，否则终端可能卡顿或输入延迟。
- 如果终端卡顿无法输入：
  1.  点击窗口左侧的三个点，尝试关闭当前终端
  2.  关不掉就直接点击 `+` 号新建一个终端
  3.  仍卡死的话，刷新整个网页即可
